NebulaFlow Cloudworks runs analytics platforms — dashboards showing supply chain risk, compliance status, and quality trends.

NebulaFlow follows the same steps as LumenDrive: browse catalogs, negotiate contracts, download documents. Same credential type. Same protocol.

The providers published their documents once. They didn't know NebulaFlow would be interested. But the catalog and credential-based policies made the documents discoverable and accessible — without any provider doing additional work.

Two consumers now access the same data for different purposes. A sixth company could join tomorrow — right credentials, same access. Providers change nothing.

### Consumer and provider are roles, not fixed labels

FerroLink is a consumer when it pulls VeloForge's material certificate. FerroLink is a provider when LumenDrive pulls its component documentation. The same company, the same participant identity, both roles.

There is no technical difference between a "consumer company" and a "provider company." Any participant can do both. The protocol and the management API are the same — the difference is which actions a company takes.
